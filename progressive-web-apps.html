Progressive web app

Progressive Web APP (PWA), aplicació mòbil nativa o aplicació híbrida?

Com veiem les tres opcions des d'Iskra.

Nativa, híbrida o PWA, què he de triar ?

Quan iniciem un projecte pensat essencialment per a dispositius mòbils, el primer que hem de fer és fer-nos aquesta pregunta: quina tecnologia he d’utilitzar?

L’estat actual del ventall tecnològic és (com a mínim) el següent:

  • - Implementació nativa amb Swift (o objective C) per iOS i Kotlin (o Java) per a Android.
  • - React-Native, amb Javascript i biblioteques de codi obert.
  • - Aplicacions web progressives, amb tecnologies web estàndard com Javascript, HTML i CSS.
Pros i contres de les diverses opcions

Fem una ullada als pros i contres de les diverses opcions que tenim i així intentar tenir una visió més clara de tot plegat.

Desenvolupament natiu

Al món dels telèfons intel·ligents, fa una bona temporada, crear aplicacions natives era l’única manera d’instal·lar una aplicació en un telèfon.

Android i iOS (sí, Windows Phone ha mort) han evolucionat de tal manera que el desenvolupament d’una aplicació nativa fa fàcil d’interactuar amb totes les funcions integrades que ofereix el maquinari del telèfon mòbil.

Per això, les aplicacions natives presenten alguns avantatges clars:

-És fàcil interactuar amb el maquinari.

Podrem utilitzar totes les característiques que el telèfon tingui. Com que parles “el seu idioma”, tindrem un millor comportament i rendiment a l’aplicació. Això implica:

  • - Desenvolupar l’aplicació per a cada plataforma
  • - Saber programar bé amb les dues plataformes (iOs, Android)
Aplicacions “híbrides”

Softwares com “Ionic” permeten, a través d’un sol desenvolupament, crear aplicacions per a diversos dispositius:

Les aplicacions hibrides tenen alguns avantatges:

  • - Només s’ha de desenvolupar una vegada (un “idioma”) i funciona en totes les plataformes.
  • - Es poden arribar a utilitzar moltes de les característiques dels telèfons (GPS, notificacions, giroscopi, NFC…)
Malgrat aquests avantatges evidents, cal dir que les aplicacions hibrides no tenen tanta fluïdesa com les aplicacions natives i que sovint (per no dir sempre) són més lentes. Això acaba donant una mala experiència d’usuari.


A més, per aplicacions que requereixen molta lògica, o projectes que es preveu que evolucionaran molt, mantenir una aplicació híbrida pot arribar a esdevenir prou complex.


Però no tot és negatiu, les aplicacions híbrides tenen coses positives: les aplicacions tenen aspecte completament natiu (tot i que poden ser lleugerament diferentes entre Android i iOs).


Aplicacions web progressives o Progressive web apps.

El concepte de PWA, és bastant nou.

Començarem doncs, amb una explicació senzilla d’alt nivell: una aplicació web progressiva és una aplicació on tots els elements actius (HTML, CSS, Javascript, Imatges, etc.) es mantenen al navegador, utilitzant treballadors web (service workers), que permeten utilitzar l’aplicació sense connexió a internet a més de d’afegir una drecera de l’aplicació a la pantalla d’inici del mòbil


Com és que Google (i també a Microsoft!) es va adonar que aquesta tecnologia era una bona oportunitat de mercat?


Mirem d’explicar-ho a partir d’apuntar alguns dels avantatges de les PWA:

  • - Les PWA funcionen a tot arreu (mòbil, tauleta, escriptori, nevera… ;)
  • - A partir de les PWA se’n pot generar una pàgina web (gairebé) sense cost.
  • - Tenint en compte que funcionen en tots els dispositius, si el disseny és senzill, tindrà el mateix disseny a tot arreu. A més, com que a la pràctica una PWA és una “web”, es pot personalitzar segons el lloc on s’estigui visualitzant: per exemple, si es visualitza des d’un iPhone, pots posar els botons a la part superior dreta, però si és un Android posar un botó flotant a la part inferior posa el botó “Crear” a la part superior dreta i si és un Android, posar el botó flotant rodó a la part inferior dreta.
  • - Es pot fer una personalització radical de l’estil gràfic.
Així doncs, quins són els problemes de les PWA?

El gran problema que tenen les PWA és que el navegador encapsula l’aplicació i, per tant, limita l’accés a les característiques del telèfon (giroscopi, bluetooth, etc…).


Malgrat aquesta limitació, les PWA aprofiten la potència dels navegadors mòbils per accedir a la geolocalització, micròfon, càmera, alguns sensors, etc… Això sí, les PWA funcionen utilitzant el Google Chrome i això vol dir dir “adéu” al Safari.


Si estàs convençut que les característiques del teu projecte encaixen amb les funcionalitats que et pot donar una PWA… un cop la tinguis no cal fer pràcticament res: no cal passar ni per l’APP Store ni per Google Play. Només s’ha de passar la PWA a producció i ja començar a funcionar.


Per tant i com a resum global

Aplicació nativa: té un cost de desenvolupament elevat (1 aplicació per plataforma), és eficient (idioma natiu) i pot aprofitar les característiques del dispositiu (telèfon).

Aplicació híbrida: molt menys cost de desenvolupament (sempre i que no es necessitin utilitzar funcionalitats pròpies del dispositiu), és bastant eficient (si es necessita més eficiència, s’ha d’utilitzar llenguatge natiu) i té característiques que poden fer que s’aproximi a les aplicacions natives (pot utilitzar l’idioma natiu si és necessari).

Una PWA: té cost de desenvolupament més baix que les aplicacions natives. Segons les característiques de l’aplicació és bastant eficient però té característiques restringides (depèn del navegador).

Com hem de decidir quina és la millor opció?

A partir de 3 preguntes claus:


Quin pressupost tenim?

Per ser clar, les aplicacions natives tenen un cost alt. S’han de desenvolupar dues vegades. Això no vol dir que les altres solucions siguin el doble de barates.


Realment necessitem ser present a les “stores”?

Aquesta és una pregunta empresarial. Si realment creiem que necessitem ser a les diverses “stores”, ja podem dir adéu a una PWA


Necessitem fer alguna cosa amb el maquinari del telèfon?

Això és més una pregunta tècnica. Si volem una aplicació amb una llista, un xat, un mapa o fins i tot una videotrucada, una PWA és perfecte. Si necessitem utilitzar més maquinari (bluetooth, giroscopi, acceleròmetre…) les PWA no són ara per ara la solució adequada.


Conclusió

Per resumir-ho, la implementació nativa us permet accedir a més funcions del telèfon, amb una millor qualitat i amb un cost superior, mentre que les PWA us proporcionaran utilitzar les funcions essencials del telèfon a un cost inferior. L’aplicació híbrida és a mig camí de les dues opcions.
